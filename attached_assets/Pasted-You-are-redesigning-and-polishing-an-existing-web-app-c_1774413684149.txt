You are redesigning and polishing an existing web app called Lucky in one coordinated pass.

Project context:
- Lucky is a Discord bot admin dashboard.
- Stack: React + TypeScript + Vite + Tailwind-style utility classes + shared UI primitives.
- The app is used by Discord server admins to manage moderation, automations, music, embeds, levels, starboard, and integrations.
- This is not a marketing site. It is a serious operations product for managing live Discord communities.

Core goal:
Do a full UI/UX refinement and product polish pass that improves both:
1. design quality and product identity
2. UX bugs, accessibility issues, mobile issues, and interaction rough edges

The result must feel:
- clean
- professional
- modern
- solid
- distinctive
- not generic
- not like a typical AI-generated dashboard
- still intuitive and operationally clear

Critical design direction:
- Do NOT turn this into a bland SaaS template.
- Do NOT use overdone AI-dashboard tropes like soft glass everywhere, generic card grids, random purple gradients, or overly rounded panels everywhere.
- Keep the product dark-first and premium.
- Preserve Lucky’s brand DNA:
  - premium, playful, high-contrast
  - purple + gold identity
  - `Sora` for display moments
  - `Manrope` for UI/body
  - `JetBrains Mono` for IDs, timestamps, and technical metadata
- Shift the visual system toward a more refined “Discord operations console” feel:
  - darker neutral surfaces
  - more intentional panel structure
  - clearer hierarchy
  - stronger navigation identity
  - more editorial composition
  - restrained but polished motion

Design references:
Use Discord bot web panels as category reference points, especially products like Carl-bot, MEE6, Dyno, and similar admin dashboards. Do NOT copy them directly. Instead, study what makes them feel purpose-built for server admins:
- operational clarity
- strong server context
- modular task organization
- fast scanning
- bot-native workflows

Your task:
Perform one combined pass that includes all of the following:

1. Design refinement
- upgrade the visual language so it feels more premium, cohesive, and distinctive
- improve layout composition, typography hierarchy, spacing rhythm, borders, panel structure, and navigation tone
- reduce generic dashboard feel
- give each major screen a stronger identity while keeping the system coherent

2. UX improvements
- improve information architecture and screen clarity
- make long settings/configuration flows easier to navigate
- reduce cognitive load
- improve admin efficiency and confidence

3. Bug and polish fixes
- fix UX bugs, inconsistent interactions, weak states, awkward flows, and rough UI details
- unify modal/dialog behavior
- improve destructive actions
- improve loading, empty, error, and save states
- improve responsive/mobile behavior
- improve keyboard and accessibility behavior

Priority issues to solve:
1. Server context clarity
- The active guild/server must always be obvious.
- Switching servers must be easy and unambiguous.
- Any “Manage” or “Open” action must always enter the correct server context.

2. Navigation and shell quality
- Make the sidebar and page shell feel like a real guild operations console, not a generic template app shell.
- Improve grouping, hierarchy, and active state treatment.
- Strengthen page headers and local context.

3. Generic dashboard composition
- Reduce repetitive card-grid layouts.
- Use more purposeful panels and task-oriented grouping.
- Create better visual hierarchy between summary, action, warning, and detail content.

4. Long admin pages
- Settings and automation pages should feel guided, not dumped.
- Add section grouping, local navigation, sticky subsections, or jump links where appropriate.
- Save behavior should feel stable and clear.
- Mobile save bars must not cover content.

5. Accessibility and interaction quality
- Use semantic buttons/links.
- Replace weak custom modal behavior with accessible dialog patterns.
- Ensure focus states, keyboard navigation, escape handling, labels, touch targets, and reduced-motion support.
- Add `color-scheme: dark` and proper dark-mode polish where needed.

6. Mobile responsiveness
- Important admin flows must work cleanly on phones and tablets.
- Avoid cramped desktop-first layouts.
- Convert dense tables/lists into clear mobile patterns with explicit labels.

7. Human-friendly admin controls
- Replace raw technical inputs like channel IDs with searchable selectors or pickers wherever possible.
- Reduce admin error risk.
- Make destructive actions safer with confirm or undo patterns.

8. Screen-specific polish
Improve these areas especially:
- Login
- Sidebar / app shell
- Servers page
- Dashboard overview
- Server Settings
- Moderation
- Auto Messages
- Embed Builder
- Music

Desired screen character:
- Login: premium and confident, but connected to the product’s operational identity
- Servers page: guild roster / readiness board
- Dashboard overview: command center
- Moderation: incident console
- Server Settings: configuration workspace
- Auto Messages: automation scheduler
- Embed Builder: creative studio
- Music: live media operations deck

Implementation constraints:
- Preserve the existing product brand and core architecture.
- Do not introduce random aesthetic changes with no UX reason.
- Prefer reusable components and systemized patterns over one-off styling.
- Keep the UI production-ready and realistic.
- Make decisions that improve both visual quality and usability at the same time.

Deliverables:
1. A prioritized audit of design, UX, bug, and polish issues
2. A redesign strategy for the main screens
3. Concrete component/system improvements
4. Specific fixes for usability, responsiveness, accessibility, and consistency
5. A phased implementation plan
6. Updated code/components/styles implementing the combined redesign + polish pass

Be specific, opinionated, and practical.
Do not give generic design advice.
Treat this as a real product redesign and frontend quality upgrade, not a cosmetic theme refresh.
